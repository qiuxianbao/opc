# Spec 技术趋势周度深度简报 工作计划
## 1. 核心目标
### 目标概述
持续跟踪大模型、Agent Skill、OpenSpec等技术前沿动态，每周交付深度洞察报告，为用户提供专属情报分析支持。

### 背景与动机
用户希望建立专属情报分析伙伴，获取中立客观的技术趋势分析，重点关注技术实现细节，辅助决策与学习。

## 2. 要求与标准
* 产出要求：
	* 周度深度报告：每周五下午交付，每条摘要100字以内，包含5-8条深度摘要，格式为Markdown（优先）或PDF。
	* 信源监测基线：配置公众号、社区、公司官网等信源列表，定期扫描更新。
* 风格偏好：
	* 中立客观，侧重技术实现细节（如架构、代码示例），暂不深入商业应用洞察。
	* 报告结构固定四个栏目：技术突破、工具更新、行业动态、趋势研判。
* 文件目录规则：
	* 过程数据与缓存：temp/
	* 核心交付资产：outputs/周报/
	* 逻辑脚本与工具：scripts/
* 数据依赖：
	* 信源列表：
		* 公众号：阿里云开发者、刘润、Datawhale
		* 社区：GitHub Trending（大模型/Agent标签）、Coze官网博客
		* 公司官网：待用户补充3-5家重点关注企业（如OpenAI、Anthropic、DeepSeek等）
	* 搜索关键词：大模型、Agent、OpenSpec、Agent Skill、技术趋势
## 3. 执行路线图
### 第一阶段：初始化
- [ ] 建立监测基线：配置信源列表，设定搜索关键词，创建初始监测脚本。
- [ ] 生成首期报告：基于已有公众号和社区信源，输出包含5-8条深度摘要的周度简报，保存至 outputs/周报/首期技术趋势简报.md。
### 第二阶段：日常运行
- [ ] 每周信息收集：周一至周四扫描信源，积累素材。
- [ ] 每周五报告生成：筛选有价值内容，按四个栏目撰写摘要，生成周度报告。
- [ ] 每周报告交付：每周五下午交付Markdown报告，可选生成PDF版本。
### 第三阶段：复盘与优化
- [ ] 每周复盘：根据用户反馈调整信源权重和筛选角度。
- [ ] 每月优化：评估报告质量，优化监测脚本和报告模板。

示例：基于首期简报生成PPT演示文稿

基于现有的首期技术趋势周度深度简报（Markdown文件），生成一份专业的PPT演示文稿。 

**输入文件**：
`outputs/周报/首期技术趋势简报.md`，该文件包含以下结构：标题、日期、四个固定栏目（技术突破、工具更新、行业动态、趋势研判）、信源说明和下期预告。每个栏目下包含若干条技术摘要，每条摘要控制在100字以内，侧重技术实现细节且保持中立客观。 

**输出要求**： 
1. 生成一份信息图风格（A.信息图风）的PPT，适合正式汇报场景。 
2. 页数结构：封面+7-9页内容+结尾，总计9-11页。 
3. 内容需涵盖简报中的所有核心信息，包括四个栏目的全部摘要。 
4. 布局应多样化，避免连续两页使用相同布局类型，交替使用矩阵、左文右图、列表详情、流程图、左右对比等布局。 
5. 每页标题应带有观点（如“技术突破：万亿参数模型推动推理效率革新”），而非纯描述。 
6. 内容要详实，每个要点都要有具体说明，避免只列大纲。 
7. 核心数据、重要概念用强调色标注。 
8. 封面页需包含主标题“首期技术趋势周度深度简报”、副标题“2026年1月29日”以及适当装饰。 
9. 结尾页包含感谢语或总结性金句。 

**执行步骤**： 
1. 读取输入文件，提取所有内容。 
2. 按照create-ppt技能规范，确定PPT风格为信息图风，查阅对应reference文件获取视觉风格描述。 
3. 规划PPT结构，将四个栏目的内容合理分配到各页面，确保逻辑连贯。 
4. 为每页编写融合式prompt，包含视觉风格描述和画面描述，文字内容用引号包裹。 
5. 调用generate_batch.py脚本生成PPT文件。 
6. 如有失败页面，按技能规范进行自动重试（最多2次），最终必须交付完整PPT文件。 

**交付物**：将生成的PPT保存为`outputs/周报/首期技术趋势简报.pptx.html`。
