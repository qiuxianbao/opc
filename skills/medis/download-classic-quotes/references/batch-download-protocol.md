# 连续下载任务执行规范

## 任务特征
用户要求连续下载多部作品的经典台词，典型表达：
- "帮我把 A 下载下来，成功之后，把 B 也下载下来"
- "先下载 X，然后再下载 Y 和 Z"
- "把...都下载下来"

## 执行步骤

### 1. 任务拆解
将连续任务拆分为独立子任务，按顺序执行：
```
任务 1: 下载《大话西游》台词
任务 2: 下载《阿 Q 正传》台词
```

### 2. 分步执行
对每个子任务：
1. 执行 `web_search` 搜索台词
2. 执行 `web_extract` 提取内容
3. 执行 `execute_code` 保存文件
4. **立即反馈该步完成情况**（文件名、大小、行数）

### 3. 进度透明
每完成一步都明确告知用户：
- ✅ 第 X 步完成
- 保存位置
- 文件大小
- 下一步计划

### 4. 最终汇总
所有任务完成后，提供汇总表格：

| 序号 | 作品 | 作者/导演 | 文件名 | 大小 | 行数 |
|------|------|-----------|--------|------|------|
| 1 | 《大话西游》 | 周星驰 | `周星驰_大话西游.md` | 6,445 字节 | 127 行 |
| 2 | 《阿 Q 正传》 | 鲁迅 | `鲁迅_阿 Q 正传.md` | 9,415 字节 | 191 行 |

并提供：
- 📁 所有文件的完整路径
- 📊 内容简介摘要
- 询问是否需要继续下载其他作品

## 关键原则

1. **不要合并任务**：即使都是下载，也要分开执行，确保每步都可验证
2. **及时反馈**：每步完成后立即反馈，不要等所有任务完成再统一反馈
3. **保持一致性**：文件命名、内容结构、格式保持统一
4. **允许中断**：如果中途用户有新指示，可以暂停当前任务

## 错误处理

- **某一步失败**：告知用户第 X 步失败，询问是否继续或重试
- **找不到作品**：明确告知哪部作品找不到，跳过该部继续后续任务
- **用户中断**：保存当前进度，询问如何处理剩余任务
